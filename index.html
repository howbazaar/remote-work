Something to work on

I wanted us to have something more than just a "hello world" project to work on. So I decided to use Lingo.

So we can play without annoying Jesse too much, I forked a copy for Code Craft.

github.com/codecraft-dunedin/lingo

Grabbing your code

  • Get some directory where you are going to keep your source files
  • git clone git@github.com:<you>/lingo
  • This makes your copy the origin
  • Only one branch: master

Making a branch for your changes

  • Generally you don't want to make changes on your copy of master. You want that to reflect the agreed on true master.
  • git checkout -b hacking
  • This makes a branch off the current one (master), and changes to it.

Make some changes.

  • Edit some files.
  • Commit the changes.

Proposing your changes

  • Pushing changes to github
  • Creating a pull request
  • Having the code review / conversation

Pushing changes

  • git push origin HEAD
  • This means, push the head of the current branch to the origin remote into a branch of the same name.
  • Yes there are other ways, but lets start here

New pull request

  • Check the base branch
  • If this is just playing around, choose codecraft-dunedin/lingo
  • If this is a real change for Jesse's project, choose lingo-reviews/lingo

Grabbing the updated copy

  • Add a remote for the 'master'
  • git remote add upstream git@github.com/codecraft-dunedin/lingo.git

pulling master

  • git checkout master
  • git pull upstream master
  • git branch -d hacking
